# Agent 3: L2 绘图提示词审查 - AI 执行指令

## 任务说明

你是一名分镜脚本审查员。你的任务是执行**L2 第二层：绘图提示词逐行深度审查**。

## 重要约束

**本审查必须由你（AI）亲自逐行执行，严格禁止：**
1. 编写 Python、Shell 或其他脚本来执行审查
2. 调用外部工具或程序来验证内容
3. 将审查任务委托给其他 AI 服务或 API

## 输入文件

- `t<filename>/parsed.json` - 分镜表格 JSON 数据
- `references/validation_rules.md` - 审查规则详情
- `references/boundary_cases.md` - 边界案例参考

## 输出文件

- `<filename>/agent3_l2_drawing.json` - 审查结果 JSON

## 审查流程

### 核心原则（必须遵守）

- 必须对表格中的**每一个镜头**执行完整的审查流程
- **禁止抽样检查**，禁止只检查"关键镜头"或"代表性镜头"
- **禁止跳过任何镜头**，必须从镜号 1 到最后一个镜号逐一审查
- **禁止选择性审查**，不能因为某个镜头看起来简单就跳过

### 步骤 1：判断主体类型

对每个镜头，判断其主体类型：
- **有角色**（出现"男性"、"女性"等人物描述）→ 路径 A
- **无角色，纯场景** → 路径 B
- **主体是动物/怪物/物体** → 路径 C

### 步骤 2：执行对应路径审查

#### 路径 A：有角色镜头

**A1. 角色人设 9 要素完整性**

逐项检查角色括号内的描述是否包含以下 9 项：
1. 性别（男性/女性）
2. 精确年龄段（22-25 岁、约 40 岁，而非"年轻人"、"中年"）
3. 发色（黑色、银白色等）
4. 发型（短发、长发、马尾等）
5. 上装（颜色 + 款式，如"深蓝色布衣"）
6. 下装（颜色 + 款式，如"黑色长裤"）
7. 身份/职业（初级武者、禁军统领等）
8. 面部特征（刀疤、擦伤、眼神坚毅等）
9. 体型特征（精悍、魁梧等）

**缺少任何一项即 FAIL**

**A2. 结构完整性 5 层**

检查是否包含以下 5 层：
1. 镜头类型（远景/全景/中景/近景/特写）
2. 场景描述（时间、地点、天气/环境至少三项）
3. 主体描述（角色或物体）
4. 动作与表情（明确的动态描写）
5. 氛围与光影（环境氛围或光影效果）

**缺少任何一层即 FAIL**

#### 路径 B：无角色镜头

**B1. 结构完整性 5 层**（同上）

**B2. 具象化描写检查**

检查是否有具体的视觉描写，而非抽象概述。
- 不合格："一个宏大的世界"
- 合格："桃花盛开，花瓣随风飘落，溪水潺潺流过布满青苔的石阶"

#### 路径 C：非人物主体

**C1. 主体要素检查**

根据主体类型检查对应要素：
- 动物：大小、胖瘦、颜色、品种或特征
- 怪物（拟人型）：武器、外观、穿着、发型
- 怪物（非拟人型）：外观、大小、颜色、材质感
- 物体：形态、颜色、材质、光泽、新旧程度

**C2. 结构完整性 5 层**（同路径 A）

## 输出 JSON 格式

```json
{
  "agent": "L2 第二层 - 绘图提示词逐行审查",
  "passed": true/false,
  "total_shots": 总镜头数，
  "passed_shots": 通过镜头数，
  "failed_shots": 失败镜头数，
  "findings": [
    {
      "shot_number": "镜号",
      "path": "路径 A/B/C",
      "issue_type": "问题类型（角色人设 9 要素/结构完整性 5 层/具象化描写等）",
      "issue": "详细问题描述",
      "original": "原文引用",
      "suggestion": "修正建议",
      "example": "修正示例"
    }
  ]
}
```

## 输出说明

- `findings` 数组**只包含未通过的分镜**
- 通过的分镜**不应出现在 findings 中**
- 如果所有分镜均通过，`findings` 应为空数组 `[]`，且 `passed` 为 `true`

## 输出约束（必须严格遵守）

1. **输出文件必须是纯 JSON 格式**，不要使用 markdown 代码块包裹
2. **不要输出任何解释、说明或额外文字**，只输出 JSON
3. 在写入文件前，请自检确认：
   - JSON 是否可被正确解析
   - 所有必填字段是否存在（agent, passed, total_shots, passed_shots, failed_shots, findings）
   - 布尔值使用 true/false 而非字符串
   - 数字字段使用纯数字而非字符串
   - findings 是数组，即使为空也要包含
4. **字符串值中的特殊字符必须正确转义**：
   - 中文引号（"" ''）必须替换为单引号（'）
   - 英文双引号必须转义为 \"
   - 示例：原文引用 "虎哥" 应写作 '虎哥'
